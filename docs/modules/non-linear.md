# 📘 **Nonlinear Equations Module — UI & Execution Guide**

## 1. Overview

The **Nonlinear Equations Module** of NA‑Engine allows users to find roots of single‑variable nonlinear functions using classical iterative methods:

- **Bisection**  
- **False Position (Regula Falsi)**  
- **Newton**  
- **Secant**  
- **Fixed Point**

The module uses function‑based input (`f(x)`), initial guesses and/or intervals, and produces:

- root approximation  
- iteration table  
- symbolic expression of the method  
- convergence plot  
- UI blocks generated by `UIContract`

This document describes the full module workflow, internal architecture, and practical recommendations.

---

## 2. UI Workflow

The complete workflow follows this pattern:

```
Select method → Provide f(x) → Provide x0/x1/interval → Run → View results
```

Each method requires different inputs, but the overall flow remains consistent.

---

## 3. Step 1 — Select Nonlinear Method

The user selects one of the available methods:

| Method | Requires | Description |
|--------|----------|-------------|
| **Bisection** | interval [a, b] | Robust method based on sign change |
| **False Position** | interval [a, b] | Faster variant of bisection |
| **Newton** | x0 | Fast method based on derivatives |
| **Secant** | x0, x1 | Derivative‑free method similar to Newton |
| **Fixed Point** | x0, g(x) | Direct iteration x = g(x) |

The selected method determines:

- visible inputs  
- method‑specific validations  
- symbolic expression  
- executor behavior  

---

## 4. Step 2 — Provide Function Input

The module uses a single input mode:

### **Function Mode**

The user provides:

- `f(x)` — Python‑style function (`x**2 - 5`, `sin(x) + x**3`)  
- `x0` — initial guess (Newton, Secant, Fixed Point)  
- `x1` — second initial guess (Secant)  
- `interval = [a, b]` — interval (Bisection, False Position)  
- `g(x)` — iteration function (Fixed Point)

The constructor:

1. converts the function to a symbolic expression using SymPy  
2. generates `f(x)` via `lambdify`  
3. generates `f'(x)` when required  
4. generates `g(x)` and `g'(x)` for fixed point  
5. validates tolerance, iteration count, and input structure  

---

## 5. Step 3 — Data Loading

Unlike Integration or Derivatives, this module **does not build DataFrames in the callback**.

The **executor** generates:

- list of iterations `x_n`  
- list of values `f(x_n)`  
- final iteration table (`pd.DataFrame`)  
- symbolic expression of the method  

This keeps the callback lightweight and consistent with NA‑Engine’s architecture.

---

## 6. Step 4 — Run Nonlinear Solver

When the user clicks **Run**, the callback:

1. builds `input_data` according to the method  
2. creates a `NumericalMethod("non_linear_equation")` object  
3. runs `validate_input()`  
4. runs `execute()`  
5. passes the result to `UIContract`  
6. renders the blocks in `nonlin-result-area`

---

## 7. Rendered Results

The dynamic result area displays:

### 7.1 Numerical Result
```
f(x*) ≈ 0
x* ≈ ⧼value⧽
```

---

### 7.2 Symbolic Expression

Examples:

- Newton: `x_{n+1} = x_n - f(x_n)/f'(x_n)`  
- Secant: `x_{n+1} = x_n - f(x_n)(x_n - x_{n-1})/(f(x_n) - f(x_{n-1}))`  
- Bisection: `c = (a + b)/2`  
- False Position: `c = b - f(b)(b - a)/(f(b) - f(a))`  
- Fixed Point: `x_{n+1} = g(x_n)`  

---

### 7.3 Iteration Table

Generated by the executor:

| iter | x_n | f(x_n) |
|------|-----|---------|
| 1 | 2.0 | -1.0 |
| 2 | 2.236 | 0.01 |
| … | … | … |

---

### 7.4 Plot

Generated by UIContract:

- iteration points  
- convergence toward the root  
- optionally f(x) over the interval  

---

### 7.5 Error Blocks

In case of error:

- `ValidationError` → incorrect input  
- `ExecutionError` → divergence, zero derivative, zero denominator  
- clear messages and input context  

---

## 8. Internal Architecture

The module uses the full NA‑Engine pipeline:

```
UI → Callbacks → NumericalMethod → Constructor → Validator → Executor → Renderer → UIContract → Dash Components
```

### Key Components

- `nonlinear_callbacks.py` — UI logic  
- `NonLinearEquation` — constructor  
- `NonLinearValidator` — validation  
- `NonLinearExecutor` — numerical computation  
- `Renderer` — payload normalization  
- `UIContract` — visual block construction  
- `result_view.py` — final rendering  

---

## 9. Practical Recommendations per Method

### **Bisection**
| Property | Recommendation |
|-----------|---------------|
| Interval | Must satisfy f(a)*f(b) < 0 |
| Tolerance | 1e‑6 |
| Iterations | 50–100 |
| Notes | Most robust method |

---

### **False Position**
| Property | Recommendation |
|-----------|---------------|
| Interval | Same as bisection |
| Tolerance | 1e‑6 |
| Iterations | 50 |
| Notes | Faster than bisection |

---

### **Newton**
| Property | Recommendation |
|-----------|---------------|
| x0 | near the root |
| Derivative | must not be zero |
| Notes | Very fast but may diverge |

---

### **Secant**
| Property | Recommendation |
|-----------|---------------|
| x0, x1 | must be distinct |
| Notes | Similar to Newton without derivatives |

---

### **Fixed Point**
| Property | Recommendation |
|-----------|---------------|
| g(x) | must satisfy |g'(x0)| < 1 |
| Notes | Can diverge easily |

---

## 10. Example Usage

1. Select **Newton**  
2. Enter:
   - `f(x) = x**2 - 5`
   - `x0 = 2.0`
3. Click **Run**

The system will display:

- root approximation  
- iteration table  
- symbolic expression  
- convergence plot  

---

## 11. Common Errors

| Error | Cause | Solution |
|-------|-------|----------|
| `ValidationError` | missing x0/x1/interval | complete inputs |
| `ExecutionError` | zero derivative | change x0 |
| `ExecutionError` | zero denominator (secant) | change x0, x1 |
| `ExecutionError` | g(x) diverges | choose another g(x) |
| `ExecutionError` | f(a)*f(b) > 0 | choose valid interval |

---

## 12. Extending the Module

To add new methods:

1. Add entry in `method_catalog.json`  
2. Implement constructor if new inputs are required  
3. Implement method‑specific validator  
4. Implement executor logic  
5. Add option in the layout  
6. Add callback logic  

No changes are required in:

- `NumericalMethod`  
- `Renderer`  
- `UIContract`  
- `result_view`
