# 📘 **ODE Module — UI & Execution Guide**

## 1. Overview

The **ODE Module** of NA‑Engine solves **Ordinary Differential Equations** using:

- **IVP methods**  
  - Euler  
  - Heun  
  - Runge–Kutta 2 (RK2)  
  - Runge–Kutta 4 (RK4)  
  - Adams–Bashforth 2 (AB2)  
  - Adams–Bashforth 3 (AB3)  
  - Adams–Moulton 2 (AM2)  
- **Systems of ODEs**  
  - RK4 for vector systems  
- **Boundary Value Problems (BVP)**  
  - Shooting method  
  - Finite Differences

The module uses function‑based input (`f(x, y)` or systems), initial conditions and/or boundary conditions, and produces:

- numerical solution `y(x)`  
- grid points `x`  
- method expression (symbolic)  
- optional system trajectories  
- UI blocks generated by `UIContract`.

---

## 2. UI Workflow

General flow:

```text
Select method → Select input mode → Provide f(x,y) or system → Provide IVP/BVP data → Run → View results
```

---

## 3. Step 1 — Select ODE Method

The user selects one of the available methods:

| Method                | Type  | Description                          |
|-----------------------|-------|--------------------------------------|
| Euler                 | IVP   | First‑order explicit method          |
| Heun                  | IVP   | Improved Euler (predictor‑corrector)|
| RK2                   | IVP   | Second‑order Runge–Kutta             |
| RK4                   | IVP   | Classical fourth‑order Runge–Kutta   |
| RK4 System            | System| RK4 for vector ODE systems           |
| Shooting (BVP)        | BVP   | IVP‑based BVP solver                 |
| Finite Differences    | BVP   | Grid‑based BVP solver                |
| Adams–Bashforth 2     | IVP   | Explicit multistep (2‑step)          |
| Adams–Bashforth 3     | IVP   | Explicit multistep (3‑step)          |
| Adams–Moulton 2       | IVP   | Implicit multistep (2‑step)          |

The selected method determines:

- which cards are visible (IVP, system, shooting, finite differences)  
- which inputs are required  
- which executor routine is used  
- which expression is rendered.

---

## 4. Step 2 — Input Mode & Function/System

### 4.1 Input Mode

The module supports two input modes:

- **Function mode**  
  - Single ODE: `y' = f(x, y)`  
- **System mode**  
  - System of ODEs: `y_i' = f_i(x, y₁, …, y_n)`

The layout exposes:

- `ode-function-card` — for `f(x, y)`  
- `ode-system-card` — for system definitions (textarea)  

Callbacks toggle these cards via the `hidden` property based on `ode-input-mode`.

### 4.2 Function Mode

User provides:

- `f(x, y)` — Python‑style expression (`x + y`, `np.sin(x) - y`)  

Constructor stores:

- `function` → `self.f`  

Validator ensures:

- non‑empty string  
- numeric fields (`x0`, `y0`, `x_end`, `h`) are valid.

### 4.3 System Mode

User provides:

- system of equations in textarea (one per line)  
- initial vector `y₀` in another textarea (`1, 0, -2`)

Constructor stores:

- `system` → list of function strings  
- `y0` → list of floats  

Validator ensures:

- non‑empty list of strings  
- `len(y0) == len(system)`  
- numeric fields (`x0`, `x_end`, `h`) are valid.

---

## 5. Step 3 — IVP / BVP Data

### 5.1 IVP Fields

Visible in `ode-ivp-card` for IVP methods:

- `x0` — initial x  
- `y0` — initial y  
- `x_end` — final x  
- `h` — step size  

Constructor:

- `x0`, `y0`, `x_end`, `h` → common IVP fields  

Validator:

- all are real numbers  
- `h > 0`  
- `x_end > x0`.

### 5.2 System IVP Fields

For `rk4_system`:

- `system` — list of functions  
- `y0` — initial vector  
- `x0`, `x_end`, `h` — scalar fields  

Validator:

- `system` non‑empty list of strings  
- `y0` list of reals  
- matching lengths.

### 5.3 Shooting BVP Fields

Visible in `ode-shooting-card`:

- `alpha` — boundary condition at `x0`  
- `beta` — boundary condition at `x_end`  
- `s0` — initial slope guess  
- `h` — step size  

Constructor:

- `alpha`, `beta`, `s0`, `h` → BVP fields  

Validator:

- all numeric  
- `h > 0`  
- `x_end > x0`.

### 5.4 Finite Differences BVP Fields

Visible in `ode-fd-card`:

- `alpha` — boundary at `x0`  
- `beta` — boundary at `x_end`  
- `n` — number of subdivisions  

Constructor:

- `alpha`, `beta`, `n` → BVP fields  

Validator:

- `n` integer ≥ 3  
- `x_end > x0`  
- numeric boundaries.

---

## 6. Step 4 — Execution Flow

When the user clicks **Calcular**:

1. Callback `run_ode` builds `input_data` from visible cards.  
2. Creates `NumericalMethod("ode", input_data)`.  
3. Calls `validate_input()` (ODEValidator).  
4. Calls `execute()` (ODEExecutor).  
5. Passes the outcome to `UIContract`.  
6. Renders the result in `ode-result-area`.

---

## 7. Rendered Results

### 7.1 Numerical Solution

For IVP:

- arrays `x`, `y`  
- plotted trajectory `y(x)`  

For systems:

- `x` plus `y` as list of vectors  
- multiple curves in the plot.

For BVP:

- grid `x`  
- solution `y` satisfying boundary conditions.

---

### 7.2 Symbolic Expressions

Examples:

- Euler:  
  `y_{n+1} = y_n + h f(x_n, y_n)`  
- Heun:  
  `y_{n+1} = y_n + h/2 (k1 + k2)`  
- RK4:  
  `y_{n+1} = y_n + h/6 (k1 + 2k2 + 2k3 + k4)`  
- AB2:  
  `y_{n+1} = y_n + h/2 (3f_n - f_{n-1})`  
- AB3:  
  `y_{n+1} = y_n + h/12 (23f_n - 16f_{n-1} + 5f_{n-2})`  
- AM2:  
  `y_{n+1} = y_n + h/2 (f_n + f_{n+1})`  
- Shooting:  
  IVP solver plus boundary matching.  
- Finite Differences:  
  discrete approximation of `y'' = f(x)` via linear system.

---

### 7.3 Tables & Plots

Executor provides:

- `x` and `y` arrays  
- optional system trajectories  

UIContract builds:

- line plots of `y(x)`  
- multi‑curve plots for systems  
- tables of `(x, y)` when needed.

---

### 7.4 Error Blocks

On failure:

- `ValidationError` → invalid input (missing fields, wrong types, bad domain).  
- `ExecutionError` → numerical failure (overflow, singular matrix, divergence).  

UIContract renders:

- error message  
- context (input_data)  
- consistent styling across modules.

## 8. Internal Architecture

The ODE module uses the full NA‑Engine pipeline:

```text
UI → Callbacks → NumericalMethod → ODE (Constructor) → ODEValidator → ODEExecutor → Renderer → UIContract → Dash Components
```

### Key Components

- `ode_section.py` — layout (cards, inputs, result area).  
- `ode_callbacks.py` — UI logic (show/hide cards, build input_data, run solver).  
- `ODE` — constructor (structural parsing of input_data).  
- `ODEValidator` — strict validation per calculation_mode.  
- `ODEExecutor` — numerical methods (Euler, RK, Adams, Shooting, FD).  
- `Renderer` — payload normalization.  
- `UIContract` — visual block construction.

## 9. Practical Recommendations per Method

### IVP Methods (Euler, Heun, RK2, RK4, Adams)

| Property   | Recommendation           |
|-----------|--------------------------|
| `h`       | small enough for stability |
| `x_end`   | not too far from `x0`     |
| Function  | smooth in `[x0, x_end]`   |

### Systems (RK4 System)

| Property   | Recommendation           |
|-----------|--------------------------|
| Dimension | keep moderate (≤ 10)     |
| Step `h`  | small for stiff systems  |

### Shooting

| Property   | Recommendation           |
|-----------|--------------------------|
| `s0`      | reasonable initial slope |
| `h`       | small enough             |
| BVP       | not too stiff            |

### Finite Differences

| Property   | Recommendation           |
|-----------|--------------------------|
| `n`       | large for smooth solutions |
| Matrix    | check conditioning        |

## 10. Example Usage

1. Select **Runge–Kutta 4**.  
2. Input mode: **Función f(x, y)**.  
3. Enter:
   - `f(x, y) = x + y`  
   - `x0 = 0.0`  
   - `y0 = 1.0`  
   - `x_end = 1.0`  
   - `h = 0.1`  
4. Click **Calcular**.

The system will display:

- numerical solution `y(x)`  
- plot of the trajectory  
- symbolic RK4 expression.

## 11. Common Errors

| Error            | Cause                          | Solution                      |
|------------------|--------------------------------|-------------------------------|
| `ValidationError`| missing IVP/BVP fields         | complete required inputs      |
| `ValidationError`| `h <= 0` or `x_end <= x0`      | adjust domain and step        |
| `ExecutionError` | overflow / divergence          | reduce `h`, shorten interval  |
| `ExecutionError` | singular matrix (FD)           | adjust `n` or function        |

## 12. Extending the Module

To add new ODE methods:

1. Add entry in `method_catalog.json`.  
2. Extend `ODE` constructor if new fields are needed.  
3. Add method‑specific validation in `ODEValidator`.  
4. Implement `_run_<method>()` in `ODEExecutor`.  
5. Add option in `ode_section.py` layout.  
6. Update `ode_callbacks.py` to handle visibility and input_data.

Core components (`NumericalMethod`, `Renderer`, `UIContract`) remain unchanged.
