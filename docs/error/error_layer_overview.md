# Error Layer Overview

## Introduction

The NA‑Engine *Error Layer* is responsible for intercepting, classifying, and normalizing any exception generated by the numerical modules. Its primary goal is to ensure that the UI receives consistent, safe, and easy‑to‑interpret error messages, without exposing internal tracebacks or backend details.

This layer is essential for Stage 4 because it enables:

- Uniform error presentation in the UI.
- Automatic documentation of errors.
- Easier debugging.
- Clear separation between mathematical logic and presentation.

---

## Objectives of the Error Layer

1. **Unify errors** coming from any module.
2. **Classify** each exception according to its type.
3. **Normalize** the output into a standard JSON format.
4. **Provide context** for debugging and documentation.
5. **Prevent the UI from receiving raw exceptions**.

---

## General Architecture

The Error Layer consists of:

- `ErrorNormalizer`: the class that converts exceptions into a standard dictionary.
- Integration with `NumericalMethod.execute()`.
- A catalog of error types.
- Flow diagrams describing the process.

---

## Standard Error Format

All errors follow this structure:

```json
{
  "status": "error",
  "error_type": "ValidationError",
  "message": "x0 must be a real number.",
  "context": {
    "method": "ode",
    "input_data": { ... }
  }
}
```

---

## Main Components

### 1. ErrorNormalizer

- Receives an exception.
- Determines its type using a dispatcher.
- Builds a standardized dictionary.
- Returns a safe error object for the UI.

### 2. NumericalMethod.execute()

- Executes the numerical method.
- If an exception occurs, it is passed to `ErrorNormalizer`.
- Returns a uniform JSON response.
